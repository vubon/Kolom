# Kolom Scripts

This directory contains various utility scripts for developing, building, and releasing Kolom.

## 🚀 Maintainer Release Checklist
Whenever you are ready to publish a new official update to GitHub (you should **not** do this on every single code commit, only when you are ready to publish a new version), follow these steps:

1. **Bump the Version:** Run `./Scripts/bump_version.sh <new_version>` (e.g. `1.1.0`)
2. **Test the Build:** Run `./install.sh` locally to make sure your new version compiles and installs perfectly.
3. **Commit to Git:** Commit your code changes, tag the release, and push to GitHub.
   ```bash
   git add .
   git commit -m "Release v1.1.0"
   git tag v1.1.0
   git push origin main
   git push --tags
   ```
4. **Create GitHub Release:** Upload the source code or a `.pkg` generated by `build_pkg.sh` to the GitHub Releases tab.

## Development Scripts

* **`deploy.sh`**: A local development helper. After you compile Kolom in Xcode (⌘B), run `./Scripts/deploy.sh` to automatically kill the running instance, install the new build to `~/Library/Input Methods`, and refresh the macOS Input Source caches.
* **`uninstall.sh`**: Completely removes Kolom from your system by deleting the app from all Input Method directories and refreshing the macOS text caches.
* **`generate_icons.swift`**: A Swift script that programmatically draws and exports the exact Kolom icon template (`BengaliIconTemplate.tiff`) used in the App Bundle and the high-res hero image.

## Release & Versioning Scripts

These scripts are used to safely version and package Kolom for public distribution, following [Semantic Versioning](https://semver.org/).

### `bump_version.sh`
Automatically updates the `Info.plist` with a new version number.

**Usage:**
```bash
./Scripts/bump_version.sh <new_version> [build_number]
# Example: ./Scripts/bump_version.sh 1.2.0
```
If you omit the build number, it will automatically increment the existing build number by 1.

### `build_pkg.sh`
Compiles Kolom in `Release` configuration and packages it into a macOS `.pkg` Installer that automatically handles placement in `/Library/Input Methods` and system cache clearing.

**Usage:**
```bash
# Build an unsigned test package
./Scripts/build_pkg.sh

# Build and sign for Apple Developer distribution
./Scripts/build_pkg.sh \
    --sign-app "Developer ID Application: Your Name" \
    --sign-pkg "Developer ID Installer: Your Name"
```

### `Packaging/postinstall`
An internal bash script embedded into the `.pkg` installer generated by `build_pkg.sh`. It forces macOS to recognize the newly installed Input Method immediately without requiring a reboot.
